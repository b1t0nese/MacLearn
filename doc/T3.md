# Структура и работа с проектом (в разработке)
- Работа с файлами и самим проектом будет работать через специальный модуль
- Структура каждого проекта будет такой:
```
project1/
├── images/      # изображения для обучения и тренировки
│   ├── train/
│   └── val/
├── labels/      # аннотация (может быть файл заместо папки, в будущем)
│   ├── train/
│   └── val/
├── dataset.yaml # конфиг для самой тренировки, будет базово заполняться судя по настройкам датасета
└── project.conf # конфиг для программы
```

#
# Интерфейс
- **Меню "Файл"** - стандартные операции (Открыть, Сохранить, Сохранить как, Выход)
- **Несколько вкладок** для управления проектом: Проект, Датасет, Создание
- **Окно для управления подклассом** - будет открываться при нажатии на любой подкласс или кнопку нового для изменения, настройки изображения или текста для поиска изображений в класс
- **Окно для управления каждым изображением в датасете** - будет открываться при нажатии на одно из изображений во вкладке "Датасет"

## Вкладка "Проект"

### 🏗️ Структура
- **Область классов** - основное рабочее пространство для управления классами объектов и их настройки для автоматизации создания датасета
- **Настройки датасета** - параметры генерации и экспорта данных
- **Кнопка "Сохранить проект"** - фиксация всех изменений

### Система классов объектов

## Управление классами:
- **Добавление класса** - кнопка "+ Добавить класс" создает новый класс
- **Название класса** - редактируемое поле с жирным шрифтом
- **Активация/деактивация** - чекбокс для временного отключения класса
- **Удаление класса** - кнопка "❌" справа в заголовке класса

### Структура класса:
- **GridLayout** для подклассов - автоматическое размещение и перенос на новые строки
- **Динамическое изменение размеров** - ширина зависит от количества подклассов, высота увеличивается при переносе
- **Прокрутка** - вертикальный скролл для навигации по классам

### Карточка подкласса:
- **Вся карточка кликабельна** - QPushButton с hover-эффектами, для открытия специального окна настройки
- **Область изображения** - центральная часть с текстом "Нет изображения" (заглушка), изображение будет примером для поиска изображений
- **Название подкласса** - Label снизу слева, текст для поиска изображений в этот класс
- **Кнопка удаления подкласса** - "❌" справа снизу
- **Кнопка добавления** - "+" в отдельной карточке с пунктирной границей

## ⚙️ Настройки датасета

### Группа параметров (50% ширины окна):
- **Чекбоксы**:
  - "Создание валидационных данных"
  - "Аугментированные изображения" 
  - "Аннотация" (включено по умолчанию)
- **Формат аннотации** - выпадающий список (пока только YOLO)
- **Количество изображений** - SpinBox (1-999999, шаг 100)

### 💾 Кнопка сохранения
- Сохраняет изменения в проекте в project.conf, если файла нет, но уже будет выбрана папка с данными, то создаст новый файл и запишет его.

### 🔄 Логика работы (в разработке)
1. Пользователь создает классы и подклассы объектов
2. Настраивает параметры генерации датасета
3. Сохраняет проект для дальнейшей обработки
4. Система формирует структуру проекта согласно выбранным настройкам
